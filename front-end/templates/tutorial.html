    Get Started

    This won't hurt a bit.

    1. Install Python 2.7

      If you haven't already done this, though, you'll probably have trouble writing Python logic.

      Assuming you're mostly on the front-end team, go here and do what it says. Note that if you're on a Mac, you probably already have Python. It may or may not be the correct version; be sure to check.

    2. Install the Google App Engine SDK

      No worries if you don't have experience with GAE. Just hit up this page and do what it tells you.

    3. Get Git (if not installed)

      I'm serious. You need to do this. Read up on Git if you haven't used it before.

      Download and install Git here.

      Set up an account with either GitHub or BitBucket. GitHub's been "rocking git" for longer and is therefore more popular, but BitBucket offers free unlimited *private* repositories with unlimited collaborators if you have a .edu email address (aka if you go to college). I recommend BitBucket.

    4. Download this thing.

      Go here and click "fork" in the upper right to fork the repository. This will be your application, so give it the name you want, etc. Now invite your collaborators ('invite' in bitbucket, 'admin>collaborators' in github).

      Once you're ready to move on, open command prompt (or bash or whatever) and cd (change directory, for the unititated) to the parent directory of where you'd like to keep this. For example, if you wanted it on your desktop, the command would be something like cd c:/Users/Alex/Desktop.

      Clone the repository you forked (aka, the one you just 'made') from bitbucket or github. This should be something like git clone https://bitbucket.org/username/appname.git. You'll see the url towards the top of the repository page.

      You can get the full thing with docs (basically what you're looking at) here ('get source' or 'fork' in the upper right), but you should use it more for reference than as a template. Especially if you're a beginner, I would recommend taking a look at it anyway as it was built expressly with reuse and 'teaching' in mind. We'll get to this later.

    5. Make it yours.

      Open app.yaml in your favorite text editor and change the application name to your app's name. If you don't have one, make something up with your names. It should be unique; every google app engine needs a unique name.

      In app/views.py, change all the info in the "globvars" class. If you don't want all the info auto-inserted all over the site, just get rid of it and everything in the "index" function. You'll also need to delete things all over the place (mostly the <head> tag, the footer, and the nav I think.

      If you have the time/inclination, you might also change the favicon at some point. There's an example file in front-end/media/images; edit the .psd and hit file > save for web & devices and resize it to 16x16. Overwrite the favicon.ico that's currently there.

    6. Register on GAE

      GAE has a free hosting service, where your site goes on yourappname.appspot.com. I'll assume that's enough for now, but it's pretty easy to get it on your own url (you just have to sign up for Google Apps, which is free for <10 people).

      Open the GAE SDK (Google App Engine Launcher) and hit file > Add Existing Application. Browse to your app, hit okay, and click on your app and hit "run". It'll take a few minutes to boot, but once there's a green arrow next to your app name, you're good to go. Click "browse" to see the locally hosted site!

      Go to https://appengine.google.com/ and hit "Create Application". Give it your app name.

      Click "Permissions" on the left under "Administration" to add everyone else in your team.

      Then go back to the SDK, and if you had to change your app name, click "edit", and modify the app.yaml to match the name you registered with.

      When you're ready to put something up on the interwebs, just hit deploy. It's really as easy as that.

    7. Hack!

      If you run into problems that don't seem like generic issues with the given technology (ie; standard Django troubles), check out the docs code and the guestbook app,.

      If neither of those do you any good, and googling seems futile, feel free to shoot me an email anytime and I'd be happy to get back to you asap. I'm rattray@decisioncandy.com.
